# Загород — аренда домов посуточно

Лендинг аренды загородных домов + бэкенд приёма заявок на бронирование.
Заявка одновременно уходит **в Telegram** и **на e-mail**. Дизайн и все анимации
лендинга сохранены без изменений — на бэкенд вынесена только реальная отправка формы.

- **Backend:** Node.js + Express (раздаёт статику и `POST /api/booking`)
- **E-mail:** nodemailer (SMTP)
- **Telegram:** Bot API `sendMessage` (через встроенный `fetch`, без лишних зависимостей)

## Структура

```
public/index.html   — фронтенд (вёрстка, стили, анимации, форма брони)
server/
  index.js          — Express-сервер: статика + POST /api/booking
  houses.js         — справочник домов (источник правды по ценам)
  validate.js       — серверная валидация и пересчёт суммы
  notify.js         — отправка в Telegram и на e-mail (параллельно)
  rateLimit.js      — простой rate-limit по IP
  logger.js         — лог заявок в консоль и logs/bookings.log
.env.example        — шаблон переменных окружения
```

## Установка и запуск локально

```bash
npm install
cp .env.example .env   # заполните значения (см. ниже)
npm run dev            # http://localhost:3000  (авто-перезапуск)
# или
npm start
```

Открывается лендинг с полностью сохранённым дизайном. Форма брони отправляет
данные на `/api/booking`.

> Без заполненного `.env` лендинг работает и заявка принимается (и логируется),
> но доставка в Telegram/почту не произойдёт — в ответе это будет видно в поле
> `delivery`.

## Переменные окружения

| Переменная | Назначение |
|---|---|
| `PORT` | порт сервера (по умолчанию 3000) |
| `TELEGRAM_BOT_TOKEN` | токен бота от @BotFather |
| `TELEGRAM_CHAT_ID` | ID чата, куда слать заявки |
| `SMTP_HOST` / `SMTP_PORT` | SMTP-сервер (порт 587 STARTTLS или 465 SSL) |
| `SMTP_USER` / `SMTP_PASS` | логин/пароль SMTP |
| `MAIL_FROM` | от кого письмо, напр. `Загород <noreply@zagorod.ru>` |
| `MAIL_TO` | куда падают заявки (можно несколько через запятую) |
| `RATE_LIMIT_PER_MIN` | максимум заявок с одного IP в минуту (по умолчанию 5) |

### Как получить токен бота и chat_id

1. Напишите [@BotFather](https://t.me/BotFather) → `/newbot` → получите
   `TELEGRAM_BOT_TOKEN`.
2. Начните диалог со своим ботом (или добавьте его в группу) и отправьте любое
   сообщение.
3. Узнайте `chat_id`:
   - откройте `https://api.telegram.org/bot<ТОКЕН>/getUpdates` и найдите
     `chat.id`, **или**
   - напишите [@userinfobot](https://t.me/userinfobot) — он пришлёт ваш ID.
   Для канала/группы id обычно отрицательный (напр. `-100...`).

### Как настроить SMTP

Подойдёт любой SMTP: Yandex, Mail.ru, Gmail (пароль приложения), SendGrid,
Mailgun и т.д. Пример для Яндекс.Почты:

```
SMTP_HOST=smtp.yandex.ru
SMTP_PORT=465
SMTP_USER=you@yandex.ru
SMTP_PASS=пароль_приложения
MAIL_FROM=you@yandex.ru
MAIL_TO=you@yandex.ru
```

## Деплой на Railway

1. Создайте проект из этого репозитория (Railway сам определит Node.js).
2. **Variables** — добавьте все переменные из `.env.example` (кроме `PORT`,
   его Railway задаёт сам).
3. **Start command:** `npm start` (или оставьте — сработает скрипт `start`).
4. Railway выдаст публичный URL — по нему открывается лендинг, форма работает.

Так же деплоится на любой VPS: `npm install --production`, задать переменные
окружения, запустить `npm start` под pm2/systemd.

## Антиспам и безопасность

- **Honeypot** — скрытое поле `company`. Люди его не видят; если бот его
  заполнил, заявка тихо игнорируется (на клиенте показывается «успех», на сервер
  ничего не уходит; сервер также отсекает такие запросы).
- **Rate-limit** — не больше `RATE_LIMIT_PER_MIN` заявок с одного IP в минуту.
- **Цену считает сервер.** Итоговая сумма пересчитывается из `server/houses.js` —
  значению `total`, присланному клиентом, не доверяем.
- Заявки логируются в `logs/bookings.log` — не потеряются, даже если и Telegram,
  и почта временно недоступны.

## API

`POST /api/booking` — `Content-Type: application/json`

```json
{
  "house": "ozero",
  "houseName": "У озера",
  "checkIn": "2026-08-01",
  "checkOut": "2026-08-04",
  "nights": 3,
  "guests": 4,
  "name": "Иван",
  "phone": "+7 900 123-45-67",
  "total": 36000,
  "company": ""
}
```

Ответ при успехе: `{ "ok": true, "ref": "ZG-XXXXXX", "delivery": { ... } }`
Ответ при ошибке: `{ "ok": false, "error": "текст" }` (статусы 400 / 429).

## Админ-панель через Telegram-бота

Тот же бот принимает команды от админов (список — в `ADMIN_IDS`) и позволяет менять
**цены** и **занятость дат**. Данные хранятся в JSON-файле в папке `DATA_DIR`
(на Railway — смонтированный Volume `/data`; при переезде на VPS это просто файл на диске).

Бот ведёт по шагам и перед каждым шагом даёт короткую инструкцию.

Команды (в личке боту):
- `/menu` — открыть панель с кнопками «Цены» и «Занятость».
- `/price` — выбрать дом → выбрать, что настраиваем:
  - **Базовая цена** — цена за сутки по умолчанию;
  - **Цена на даты** — спеццена на конкретные даты или диапазон (сначала даты, потом цена);
  - **Цена на месяц** — цена на весь месяц (`08 2026`, затем цена);
  - **Сбросить спеццены** — вернуть всему дому базовую цену.
  Чтобы вернуть базовую цену на выбранные даты/месяц — при вводе цены пришлите `0`.
- `/busy` — выбрать дом → «Занять» или «Освободить» → прислать даты.

**Формат дат — свободный,** главное «день месяц год», разделители любые или без них:
- одна дата: `10.08.2026`, `10 08 2026`, `10082026`;
- диапазон: `10.08.2026 - 20.08.2026` или `10.08.2026 по 20.08.2026`;
- несколько сразу — через запятую или с новой строки.

Сайт берёт актуальные цены (базовые и по датам) и занятые даты из `/api/state.js`;
итоговая сумма считается по цене каждой ночи, занятые даты в форме брони недоступны,
а сервер повторно проверяет и цену, и занятость.

### Настройка на Railway
1. Добавь к сервису **Volume**, точка монтирования `/data`.
2. Переменные окружения: `PUBLIC_URL` (адрес сайта), `TG_WEBHOOK_SECRET` (любая строка),
   `ADMIN_IDS` (id админов через запятую). Webhook бот регистрирует сам при старте.
3. Каждый админ должен один раз нажать «Старт» боту в личке.
