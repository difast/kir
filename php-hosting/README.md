# GreenPark — версия для Рег.ру (PHP + MySQL, хостинг Host-0)

Это перенос сайта с Node.js (Railway) на обычный виртуальный хостинг Рег.ру.
Дизайн и весь фронтенд не менялись — переписан только бэкенд приёма заявок и
админ-бот на **PHP + MySQL**. E-mail-доставки нет (по договорённости), заявки
уходят в **Telegram**.

## Что внутри

```
public_html/                 ← корень сайта на хостинге
  index.html, dom.html, *.js, *.css, images/…   ← ваш фронтенд (без изменений)
  .htaccess                  ← роутинг /api/* на PHP, кэш фото, gzip
  api/
    config.example.php       ← ШАБЛОН настроек (переименовать в config.php)
    index.php                ← роутер: /api/booking, /api/state.js, /api/state, webhook
    db.php                   ← подключение к MySQL + утилиты
    houses.php               ← справочник домов (цены/статус)
    dates.php                ← разбор дат «день месяц год»
    store.php                ← цены и занятость (MySQL)
    validate.php             ← проверка заявки + пересчёт суммы (мин. 5 ночей)
    telegram.php             ← отправка заявок + админ-бот (цены/занятость)
    setup-webhook.php        ← открыть 1 раз в браузере — регистрирует webhook
schema.sql                   ← таблицы MySQL + стартовые цены (импорт в phpMyAdmin)
```

## Установка на Host-0 — по шагам

### 1. База данных
1. Панель Рег.ру → **Базы данных / MySQL** → создайте базу и пользователя,
   запомните: имя базы, пользователя, пароль, хост (обычно `localhost`).
2. Откройте **phpMyAdmin** → выберите созданную базу → вкладка **Импорт** →
   загрузите файл `schema.sql` → «Вперёд». Создадутся таблицы и стартовые цены.

### 2. Файлы сайта
1. Загрузите всё содержимое `public_html/` в корень сайта на хостинге
   (через файловый менеджер панели или по FTP). Папка `images/` — тоже.
2. В папке `api/` переименуйте `config.example.php` → **`config.php`** и впишите:
   - данные базы (`name`, `user`, `pass`);
   - `telegram_bot_token` — токен бота от @BotFather;
   - `telegram_chat_ids` — куда слать заявки (id админов/канала через запятую);
   - `admin_ids` — кто может менять цены/занятость через бота;
   - `public_url` — адрес сайта, напр. `https://greenparkksochi.ru`.

### 3. Webhook Telegram
Откройте в браузере один раз:
`https://ВАШ-САЙТ/api/setup-webhook.php` — увидите «✅ Webhook установлен».
Затем напишите боту в личке **/menu** — откроется панель управления.

> Каждый раз при **смене токена** бота: поменяйте `telegram_bot_token` в
> `config.php` и снова откройте `setup-webhook.php`.

### 4. Проверка
- Откройте сайт, форму брони, выберите < 5 ночей → «Минимальный срок
  бронирования — 5 ночей», отправка блокируется.
- Оформите тестовую бронь (≥ 5 ночей) → заявка придёт в Telegram, номер
  вида `ZG-XXXXXX`, запись появится в таблице `bookings`.
- В боте: `/price` — сменить цену, `/busy` — занять/освободить даты.

## Безопасность
- `config.php` закрыт от прямого доступа в `.htaccess` (там токен и пароль БД).
- Цену итоговой заявки **считает сервер** из `houses.php` + спеццен — значению
  от клиента не доверяем.
- Honeypot (скрытое поле `company`) и rate-limit по IP отсекают ботов.
- Занятые даты недоступны в форме, и сервер повторно это проверяет.

## Отличия от версии на Railway
- Хранилище: было `data.json` → стало таблицы MySQL (`prices`, `price_dates`,
  `busy_dates`).
- Состояние диалога бота: было в памяти процесса → таблица `tg_pending`
  (PHP не держит память между запросами).
- E-mail убран; заявки идут только в Telegram.
- Фронтенд (`index.html`, `dom.html`, `booking.js`, …) — без изменений.
